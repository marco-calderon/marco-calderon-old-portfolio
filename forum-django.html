Forum Web App

Repository  Demo 
Django React GraphQL

This project allows sharing posts between users of this application, in which they can share thoughts, reply with information and vote for the most useful answer. All in a secure way to ensure data consistency and privacy between users.

It is divided into a backend part written in Django and GraphQL and a frontend in React with a custom style made by @threeinfiniteaxes.

Forum App is made in Django and GraphQL (Graphene library) to provide a scalable API for any client, web based and mobile if necessary. The persistence layer is made using PostgreSQL and Django Models. Secure login is also present in form of JWT which is supported by the front end. In addition, it manages the creation of replies to ensure data consistency on the database.

The React application uses Apollo GraphQL client to connect with the back end service. Internally it uses a cache mechanism which ensures the latest data without making unnecessary requests. This feature is provided from the same library. The security is managed under-the-hood for creating resources, such as posts or replies. For voting, it also ensures consistency to avoid multiple votes for the same user and same reply or post.

It uses styled-components library for a CSS-in-JS solution inside React. What is CSS-in-JS? It is a combination of properties similar to CSS inside JavaScript code that ultimately will be converted to plain CSS but offering the capabilities of adding functions, adapt to property changes in real time with changes of the state of the application and more. It is like giving programming features to styles that in plain CSS or SCSS are not possible to implement.

Although this makes the code highly coupled into your style, it offers many benefits - many more compared to what Iâ€™ve mentioned. It also makes the CSS and functions testable, so it is rather beneficial at the long run rather than harmful. At the beginning it takes time to adjust to this solution, and more if you come from a CSS only background, but you'll notice and get the benefits right away.

Styled-components library allows creating components in a React way while also offering to pass down properties that could change its style, without a lot of code. This also offers the option to distribute the usage of the components in the application and adapt to styles according to its context. This is the main reason I had when choosing this library.

On the other hand, it is difficult to implement responsive behavior using this approach. So a combination of CSS/SCSS with CSS-in-JS is also acceptable at this point, or using hooks for listening width changes for showing the correct version according to the screen size. This is one of the main challenges for rendering in screens other than desktop or tablet sizes.
